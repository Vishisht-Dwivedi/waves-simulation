# Wave Simulation
A canvas-based wave rendering experiment built to explore direction-independent, edge-anchored wave fields for UI backgrounds.
The goal of this project was to design a system where waves:
Can travel in any direction
Always remain anchored near the edges of the screen
Transition smoothly across orientations
Can be layered to produce organic animated backgrounds
This project eventually became the prototype environment for the animated background used in Sermocino Auth UI.

## Demo Concept
The system renders multiple translucent wave layers using:
parametric lines
directional sine fields
procedural animation

Each wave is represented as a polygon generated from a sinusoidal displacement along a baseline line.

## The Mathematical Journey
This project started with a deceptively simple question:

How do you generate waves that move in any direction but remain anchored to the edges of the screen?

Several approaches were explored before arriving at the final solution.

### Attempt 1: Geometric Construction
The first idea was purely geometric.
The plan was:
Inset the screen rectangle by a distance h.
Construct a kite-shaped envelope using lines from the inset rectangle.
Slide points along two boundary segments.
Connect the points and track the midpoints of the segments.

The hypothesis was that the locus of these midpoints would form a rectangular hyperbola, which could then be differentiated to obtain the baseline slope.
While this construction worked visually, deriving the locus algebraically proved unnecessarily complex.

#### Key Insight
Instead of tracking midpoints of line segments, the problem could be reframed:
We don't actually need a curve.
We need a family of lines parameterized by angle θ that stay close to the rectangle boundary.
This leads to the normal form of a line:
##### x cosθ + y sinθ = c(θ)
Where:
θ = direction of the wave
c(θ) = perpendicular distance from the origin to the line
The problem now becomes finding the correct c(θ).

### Final Result
If the screen is centered at the origin and the rectangle has half-width a and half-height b, the inset rectangle (distance h from the edges) has sides:
|x| ≤ a - h
|y| ≤ b - h
The correct distance function becomes:
##### c(θ) = max(|(a - h) cosθ|, |(b - h) sinθ|)
This equation produces a family of lines that:
stay at least h away from the screen edges
automatically switch between rectangle sides
remain continuous across all directions

This is actually a known concept in convex geometry called the support function of a shape.

#### Why This Works
For any convex shape, the support function gives the maximum projection of the shape along a direction.
c(θ) = max_{(x,y)∈S} (x cosθ + y sinθ)
For a rectangle, this simplifies to the formula above.
This naturally produces the piecewise behavior across diagonals that was originally handled manually.

## Wave Generation
Once the baseline is defined, waves are generated by displacing points along the line's normal direction.
Let:
Normal vector:
n = (cosθ, sinθ)

Tangent vector:
t = (-sinθ, cosθ)

A point on the baseline becomes:
##### P = center + n*c(θ) + t*s
Where:
s moves along the line
Wave displacement is then applied:
P_wave = P + n * Σ A_i sin(f_i s + ω_i t + φ_i)

Multiple sine waves are combined to produce organic motion.

### Rendering
Each wave is drawn as a polygon:
Sample points along the baseline.
Apply sinusoidal displacement.
Extend the polygon outward along the normal direction.
Fill the shape with a translucent color.
Layering multiple waves produces a smooth animated background.
